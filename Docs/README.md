# project horizons docs

este pacote é uma base de documentação para o projeto `project horizons csharp`.

ele foi montado para você jogar dentro da pasta `docs/` do projeto e já sair com uma estrutura decente em vez daquele clássico estado espiritual chamado “depois eu documento” kkk.

## estrutura

- `01-overview.md` — visão geral do jogo
- `02-core-loop.md` — loop principal
- `03-gameplay-systems.md` — sistemas principais
- `04-world-generation.md` — geração procedural e streaming
- `05-architecture.md` — arquitetura de código
- `06-save-load.md` — persistência
- `07-ai.md` — IA e comportamentos
- `08-ui-ux.md` — interface e feedback
- `09-performance.md` — performance, profiling e otimização
- `10-content-pipeline.md` — pipeline de conteúdo
- `11-roadmap.md` — roadmap técnico
- `12-glossary.md` — glossário
- `13-folder-structure.md` — organização de pastas
- `14-coding-guidelines.md` — padrões de código
- `15-open-questions.md` — dúvidas pendentes e decisões em aberto

## nota importante

essa documentação foi escrita como **base estruturada**, não como espelho exato do código atual.
ou seja: está pronta pra uso, mas o ideal é ajustar nomes de classes, sistemas e decisões reais do projeto.

## como usar

1. cria uma pasta `docs/` no repositório
2. joga esses arquivos lá
3. ajusta nomes concretos do projeto
4. conforme o jogo evoluir, atualiza estes docs junto com features novas

## regra de ouro

feature sem doc vira folclore.
folclore vira bug.
bug vira sofrimento.
