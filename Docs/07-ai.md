# inteligência artificial

## objetivo

entidades controladas por IA devem ser previsíveis o suficiente para serem entendidas e complexas o suficiente para serem interessantes.

## camadas possíveis

- percepção
- decisão
- navegação
- execução de ação
- reação a eventos

## percepção

entradas comuns:
- visão
- audição
- proximidade
- dano recebido
- cheiro / heat / noise / threat score, se existir

## decisão

opções de implementação:
- finite state machine
- behavior tree
- utility ai
- goal oriented action planning

escolha depende do escopo:
- simples e previsível: fsm
- ramificado e escalável: behavior tree
- decisão contextual rica: utility ai

## estados comuns

- idle
- patrol
- investigate
- chase
- attack
- flee
- return
- dead

## navegação

definir:
- navmesh ou grid?
- suporte a terreno procedural?
- custo por atualização de rota
- comportamento quando caminho falha

## design de IA

boa IA não é só “inteligente”.
ela precisa:
- comunicar intenção
- ser legível
- ser balanceável
- não parecer trapaceira

## debug

ferramentas úteis:
- gizmos de visão
- target atual
- estado atual
- threat level
- path preview
- timers e cooldowns
