# ui / ux

## objetivos

- informar sem poluir
- responder rápido às ações do jogador
- manter consistência visual
- reduzir fricção em ações repetidas

## pilares

- clareza
- hierarquia visual
- feedback imediato
- acessibilidade
- contexto

## telas e interfaces principais

- hud
- inventário
- crafting
- mapa
- menu de pausa
- configurações
- tooltips
- janelas contextuais

## feedback essencial

o jogador precisa perceber:
- tomou dano
- causou dano
- interagiu com algo
- item foi coletado
- ação falhou
- ação foi concluída
- recurso insuficiente

## princípios

- ações críticas precisam de feedback forte
- menus usados com frequência precisam ser rápidos
- tooltips devem ser úteis, não novelinhas
- evitar esconder informação importante atrás de cliques inúteis

## acessibilidade

considerar:
- escala de ui
- contraste
- remapeamento de input
- sensibilidade de câmera
- opção para reduzir efeitos visuais intensos
